# NereusSDR v0.3.2 Alpha Test: OpenHPSDR Radios (P1 & P2)

**Hi, and thank you.** This is the v0.3.2 alpha-test guide. v0.3.2 is a substantial point release on top of v0.3.1 with four pieces of work landing together: **3M-3a-iii DEXP/VOX speech processing** (VOX and DEXP work end-to-end for the first time; the DSP wasn't actually wired up in v0.3.1 even though the menus existed), a **Hermes Lite 2 maturity push** (RF/Tune Power slider rework to match the mi0bot fork, connect-time init parity, A-ATT and N2ADR fixes, ADC overload byte offset, PA fwd/rev power scaling), **PA Setup parity + safety hotfix** (Setup → PA pages rebuilt for full Thetis parity; the kernel doubles as a PA over-drive safety hotfix for high-gain finals like the ANAN-8000DLE), and a **long tail of persistence and stability fixes** (OC matrix pins, mic-jack PTT, TUN-off cold-start, audio bus reconfigure, NR3 dev-build path).

If you ran the v0.3.1 alpha test, you don't need to redo it. The v0.3.1 doc ([docs/debugging/v0.3.1-alpha-tester-smoketest.md](v0.3.1-alpha-tester-smoketest.md)) is still the right reference for the full RX surface, the connection workflow, the speech processor (TX EQ / Leveler / ALC / CFC), the filter preset store, the TX filter overlay, and the per-board PA cal grid. This v0.3.2 doc focuses only on what changed.

---

## What's new in v0.3.2 (the short list)

1. **VOX and DEXP actually work now.** v0.3.1 had the menus and toggles in place but the underlying WDSP DSP block wasn't wired up. v0.3.2 ports the DEXP DSP and connects the VOX callback to the MOX state machine. New Setup → Transmit → DEXP/VOX page mirrors Thetis 1:1; TxApplet has a dedicated VOX row; DexpPeakMeter strip on the Setup page lets you eyeball the gate live.
2. **Hermes Lite 2 maturity push.** RF Power slider 0 to 90 in steps of 6 (was 0 to 100 in steps of 1) and Tune Power slider 0 to 99 in steps of 3, both matching the mi0bot fork; per-band Tune Power grid reads in dB instead of watts on HL2. Connect-time init now matches Thetis (resolves cold-start issue #153 and SSB MOX path). A-ATT label corrected, N2ADR default flipped to True, RX attenuator max +31 dB, P1 ADC overload byte offset (#176), HL2 PA forward and reverse power readings (16.7x over-read) corrected.
3. **PA Setup parity + safety hotfix.** Setup → PA pages rebuilt for full Thetis parity: PA Gain page with 14-band gain grid + 14×9 drive-step adjust matrix + auto-cal sweep state machine; Watt Meter page with new toggles; PA Values page closes the 4-label gap from v0.3.1. The kernel of the work is also a PA over-drive safety hotfix for high-gain finals (notably the ANAN-8000DLE) at low TUNE-slider positions. K2GX measured over 300 W out on a 200 W radio at slider 50% pre-hotfix; v0.3.2 ports Thetis's per-band PA gain table and the audio-volume math behind it. Lower-power radios (HL2, Hermes, Hermes II, Angelia, Orion) were never at risk.
4. **Persistence and stability fixes.** Per-band TX RF Power persists across band changes and restart (#192). OC matrix pins persist (#191). Mic-jack PTT enabled at firmware by default (#182). TUN-off cold-start fix (#177). Audio bus reconfigure dropped on Settings open (#172). NR3 dev-build model path (#184, thanks **KM4BLG**). Plus step-attenuator MOX gating, SWR foldback topology correction (now applies to IQ scalar only, not wire byte), TX meter rescaling per-SKU.

## Things to know before you key up

**The PA-cal hotfix changes drive-byte values.** On an ANAN-8000DLE, slider 50% on 80 m used to send wire byte 127 (full drive); v0.3.2 now sends ~49. If you're used to a particular slider position producing a particular forward power on your radio, that calibration will shift. This is the fix, not a regression. Re-cal your slider expectations.

**Per-radio TX status (unchanged from v0.3.1):**

| Radio | TX safe to bench? | Notes |
|---|---|---|
| ANAN-G2 / Saturn | **Yes** (primary TX dev target) | PA-cal hotfix verified by JJ on G2 |
| ANAN-7000DLE / 8000DLE | **Yes, bench reports especially welcome** | The PA-cal hotfix specifically targets these. K2GX field-tested on 8000DLE. |
| ANAN-200D / 100D / 100B / 100 / 10 / 10E | Yes | Watch ADC overload badge |
| Hermes / Hermes II / Angelia / Orion / Orion MkII | Yes | Bench reports welcome |
| **Hermes Lite 2** | **Yes, bench-cleared in v0.3.1** | New RF / Tune Power slider encoding in v0.3.2. See HL2 section below. |

**HL2 owners, the slider behavior changed.** RF Power slider is now 0 to 90 in steps of 6 (was 0 to 100 in steps of 1). Tune Power slider is 0 to 99 in steps of 3. Per-band Tune Power grid reads in dB instead of watts. Existing per-band Tune Power values reinterpret in place; nothing on disk needs migration. If your slider position seems to give a different power output than before, that's the encoding change, not a bug. Re-cal on a wattmeter.

**Standard items (carried over):**

- ADC Overload badge (yellow on any clip, red on level >3) is your friend on TX.
- TUNE button kicks a single-tone carrier.
- VOX is now functional. If you don't want VOX, leave it off in the TxApplet VOX row before plugging your mic in. Ambient room noise can MOX the radio if the threshold is loose.

## Before you start

### Things you need

- A supported OpenHPSDR radio on your network
- The platform-appropriate v0.3.2 build from <https://github.com/boydsoftprez/NereusSDR/releases/tag/v0.3.2>
- A mic into the radio's mic input (or use the Pc/Composite source via VAX)
- A wattmeter if you want to verify the PA-cal hotfix numerically

### Get the build

Same download flow as v0.3.1. Replace the version number in the asset names:

- macOS Apple Silicon: `NereusSDR-0.3.2-macOS-apple-silicon.dmg` (signed + notarized)
- macOS Intel: `NereusSDR-0.3.2-macOS-intel.dmg` (signed + notarized)
- Linux x86_64: `NereusSDR-0.3.2-x86_64.AppImage`
- Linux aarch64: `NereusSDR-0.3.2-aarch64.AppImage`
- Windows x64 installer: `NereusSDR-0.3.2-Windows-x64-setup.exe`
- Windows x64 portable: `NereusSDR-0.3.2-Windows-x64-portable.zip`

Verify GPG signatures against `SHA256SUMS.txt.asc` as before.

### Existing settings carry forward

No migration step needed. Saved radios, mic profiles, DSP settings, PA cal-points, per-band tune power, OC matrix entries, container layout, spectrum / waterfall settings all carry forward exactly. PA Gain profiles seed automatically on first connect.

## The actual test

### 1. VOX and DEXP

VOX and DEXP work end-to-end for the first time in v0.3.2.

1. Open Setup → Transmit → **DEXP/VOX**. The page should mirror Thetis: three group boxes (`grpDEXPVOX`, `grpDEXPLookAhead`, AMSQ NereusSDR-spin section), 2×2 grid layout, top-justified.
2. The **DexpPeakMeter** strip should be visible on the page. With your mic plugged in, you should see the peak strip respond to your voice in real time even when not transmitting.
3. Set a VOX threshold above your room noise, below your speaking level (the peak meter helps you eyeball this).
4. Make sure VOX is enabled (TxApplet VOX row → ON).
5. Speak into the mic. The radio should key up automatically (MOX should engage, the spectrum should show TX, forward power should register).
6. Stop speaking. The radio should drop out of TX after the configured release time.
7. Test DEXP: enable DEXP on the same page. Below the threshold, mic audio is gated; above, audio passes. Speak below threshold (whisper), then above (normal voice). The transmitted audio should drop out below threshold.
8. AMSQ: enable AMSQ. Same gating but on the AM-mode squelch path. Test on AM if your mode supports it.

### 2. Hermes Lite 2: RF and Tune Power encoding (HL2 owners only)

The HL2 RF Power and Tune Power sliders use new encodings that match the mi0bot fork.

1. Connect to your HL2.
2. On the **TxApplet**, the RF Power slider should run 0 to 90 in steps of 6 (was 0 to 100 in steps of 1). The slider label should show dB on HL2 (matches mi0bot).
3. The Tune Power slider should run 0 to 99 in steps of 3 (33 sub-steps).
4. Hover the sliders. Tooltip should match Thetis upstream: "Transmit Drive: relative value, not watts unless the PA profile is configured with MAX watts at 100%".
5. Setup → Transmit → **Power**. The new "Tune" group should appear above the per-band grid: 3 drive-source radios + TX TUN Meter combo + Fixed-mode spinbox (-16.5 to 0 dB in 0.5 dB increments) + Reset Tune Power Defaults button.
6. The per-band Tune Power grid should read in **dB** on HL2 (not watts). Other radio families still read watts.
7. Key TUNE at various slider positions. Confirm forward power scales as expected.

### 3. Hermes Lite 2 stability fixes

1. **Cold-start (#153)**: power-cycle your HL2, then launch the app. Connect. The MOX path should come up cleanly without needing a reconnect. SSB voice TX should work on first try.
2. **A-ATT label and N2ADR default (#174)**: Setup → Hardware → Hermes Lite Options. Confirm the A-ATT spinbox label is correct. N2ADR filter board option should default to True (was False in earlier builds). The dead checkbox that used to sit there should be gone.
3. **RX attenuator max**: the spinbox should accept up to +31 dB (not +32 dB).
4. **ADC overload byte offset (#176)**: deliberately overdrive the front end (preamp on, attenuation off, strong-signal band). The ADC Overload badge should light up on the correct frame counts; previously a P1 byte-offset error caused intermittent false / missed indications.
5. **HL2 PA forward and reverse power readings**: the FWD and REV readouts should now read in the correct watts range. Pre-hotfix they were 16.7x over-read.

### 4. PA calibration hotfix verification (high-power radios)

This is the "why upgrade now" test for ANAN-8000DLE / Orion / OrionMkII / 7000DLE / Saturn / Anvelina Pro 3 owners.

1. Connect to the radio. Confirm v0.3.2 is running (Help → About).
2. Open Setup → PA → **PA Gain**. The page should populate with a 14-band gain spinbox grid for your active profile. New profiles seed `Default - <connected model>` automatically.
3. Switch to a low-band (80 m or 40 m) on the VFO.
4. On the TX applet, set drive to 50%.
5. Press TUNE. Note the forward power on the wattmeter.
6. Pre-hotfix on an 8000DLE at slider 50% / 80 m, you'd see well over 100 W. v0.3.2 should land in the rated-power neighborhood for the slider position (typically 50% slider equates to ~25-50 W on an 8000DLE; exact depends on PA cal).
7. Step through 25%, 50%, 75%, 100% slider. Power should scale roughly linearly. **Specifically: at 100% you should see your radio's rated max power, no more.**

**If you measure over-drive at any slider position on v0.3.2, that's a bug we want to hear about immediately.** Include the radio model, the band, the slider position, and the forward-power reading.

### 5. PA Setup pages

1. Setup → PA → **PA Gain**. Confirm the 14-band gain grid + 14×9 drive-step adjust matrix + per-band max-power column populate. Try Profile combo: New / Copy / Delete / Reset Defaults.
2. Setup → PA → **Watt Meter**. Existing PA cal-point grid + new "Show PA Values page" toggle + Reset PA Values button.
3. Setup → PA → **PA Values**. The 4-label gap from v0.3.1 (Raw FWD power, Drive, FWD voltage, REV voltage) should be closed. Running peak/min tracking on six telemetry values; in-page Reset button.
4. **Auto-cal sweep** (HF-only): on the PA Gain page, enable `chkAutoPACalibrate`. The sweep state machine should walk HF bands, key TUNE per step, read FWD via RadioStatus, and write the gain delta into the active profile.

### 6. Persistence fixes

Restart the app between each step to verify persistence.

1. **OC matrix pins (#191)**: Setup → Hardware → Hermes Lite Control → SWL → set a few pins. Restart. Verify they persist. Also try `setPinAction` per-band overrides and the Reset Defaults button.
2. **Mic-jack PTT (#182)**: Setup → Hardware → Mic Jack → toggle the PTT-disabled checkbox. The setting should persist and the wire should reflect it on the next MOX.
3. **Per-band TX RF Power (#192)**: set RF Power to 30 W on 80 m, switch to 40 m, set 50 W, switch back to 80 m. Should restore to 30 W. Restart, switch through bands, values should still be there.

### 7. TUN-off cold-start (#177)

1. Connect to a radio you've never used in this session.
2. Press TUNE quickly after connect (within the first few seconds).
3. Press TUNE again to unkey. v0.3.2 defers the TUN-off gen1 + power-restore until rxReady plus 100 ms; pre-fix this could leave the radio in a stuck state.

### 8. Audio bus reconfigure (#172)

1. Open Settings while audio is playing.
2. Audio should NOT glitch / reconfigure / pop on Settings open. Pre-fix opening Settings triggered a spurious bus reconfigure event.

### 9. NR3 dev-build model path (devs only)

Only relevant if you're running an unpackaged dev build (e.g., from `cmake --build build` without packaging).

1. Run an NR3-enabled mode with a dev build.
2. The rnnoise model should load from `<exe>/../third_party` (the build tree). Pre-fix the dev binary couldn't find the model and NR3 silently failed.

### 10. Regression sweep (carry-forward checks)

These should still work as in v0.3.1. Quick smoke pass:

- Connect via discovery + via unicast probe (VPN-reach).
- TUNE, verify forward power.
- SSB voice TX (with the new VOX off), verify intelligible audio on a second receiver.
- TX EQ + Leveler + ALC + CFC: load a factory mic profile, hear the change.
- TX filter overlay on the panadapter (orange in TX, cyan in RX).
- Per-profile TX bandwidth (TxApplet TX BW spinboxes + status label).
- ADC Overload badge during TX.
- PA voltage on MkII boards reads ~13.x V (regression watch).
- Auto-connect on launch.
- Disconnect overlay (pull Ethernet, see "DISCONNECTED" fade).

If any of these regressed in v0.3.2, that's a bug.

## Things that might look weird but aren't bugs

- **Drive-byte values on high-gain PAs are lower than v0.3.1.** That's the PA-cal hotfix. Slider 50% on 80 m / 8000DLE used to send wire byte 127; now it sends ~49. Forward power at the same slider position will be lower. This is the fix.
- **HL2 RF Power slider is 0 to 90 in steps of 6.** Was 0 to 100 in steps of 1. The slider feels coarser because it now matches the underlying 16-step output attenuator. mi0bot-Thetis behaves the same way.
- **HL2 Tune Power per-band grid reads in dB.** Was watts. Other radio families still read watts. The dB scale matches mi0bot.
- **VOX popup is gone.** The floating VOX popup and the legacy VoxDexpSetupPage have been retired. All VOX/DEXP controls are on Setup → Transmit → DEXP/VOX or the TxApplet VOX row.

## What we'd love you to try that's unusual

- **PA-cal hotfix verification on every high-gain PA we don't have direct bench access to.** ANAN-7000DLE, 8000DLE, OrionMkII, Saturn, Anvelina Pro 3. Measure forward power at slider 25/50/75/100% on a low band (80 or 40 m). Compare against your radio's rated max. **Anything over rated max is a bug we want to know about immediately.**
- **VOX threshold + room noise margin.** What range of VOX thresholds gives you comfortable hands-free SSB without false-triggering on ambient noise? The DEXP gate-ratio + look-ahead settings are highly tunable now; let us know which configurations work for you.
- **HL2 RF/Tune Power encoding in real-world QSOs.** The new slider encoding matches mi0bot exactly, but field testing is the only way to confirm it feels right. If you're a long-time HL2 user with strong opinions about the slider behavior, weigh in.
- **OC matrix on HL2.** The persistence fix in #195 covers `setPin`, `setPinAction`, and `resetDefaults`. If you have a complex SWL band x pin setup, restart the app a few times and confirm nothing drifts.
- **Long TX sessions with VOX.** The DSP is brand new in v0.3.2; long-soak testing matters. 30+ second SSB transmits with VOX engaged. Watch for thermal drift, audio dropouts, gate chatter.

What we especially want to hear about:
